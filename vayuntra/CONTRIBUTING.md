# Contributing to Vayuntra

## Prerequisites

- Read `SECURITY.md` before contributing
- All contributors must sign commits (GPG)
- Review `BRANCHING_STRATEGY.md`

## Contribution Workflow

1. Fork the repository
2. Create a feature branch from `develop`
3. Implement changes
4. Ensure all tests pass locally
5. Open a Pull Request to `develop`
6. Await review and CI/CD pass

## Code Standards

- Python: PEP 8 compliant (enforced via flake8)
- JavaScript/React: ESLint enforced
- All new backend code must include unit tests
- No secrets, credentials, or PII in code

## Pull Request Requirements

- Descriptive title following commit convention
- Summary of changes
- Test evidence
- Security impact assessment (if applicable)

## Questions

Open a Discussion on GitHub. Do not use Issues for general questions.
